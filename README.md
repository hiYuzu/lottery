# 🎉 多人同步抽奖系统

局域网内多人同步的实时抽奖系统，所有客户端状态一致，不会出现重复抽奖或状态回退。

## 快速开始

```bash
# 1. 安装依赖
npm install

# 2. 启动服务
node server.js

# 3. 浏览器访问
# 抽奖页面: http://localhost:3000
# 管理后台: http://localhost:3000/admin.html
# 局域网其他设备: http://<本机IP>:3000
```

## 功能说明

| 功能 | 说明 |
|------|------|
| 实时同步 | 所有客户端通过 WebSocket 保持状态一致 |
| 奖项管理 | 动态添加/删除/编辑奖项，支持拖拽排序 |
| 单次多人抽取 | 每个奖项可设置单次抽取人数（perDraw） |
| 阳光普照奖 | 独立奖项类型，全员可参与（不受部门限制） |
| 背景音乐 | 页面加载后点击任意位置自动播放，可随时开关 |
| 中奖动画 | 名字快速滚动 → 弹出中奖者 → 烟花庆祝 |
| 抽奖记录 | 自动记录每次抽奖的时间、奖项和中奖者 |
| 结果导出 | 一键导出中奖汇总和详细记录为文本文件 |
| 重置功能 | 清空所有中奖记录和抽奖历史，重新开始 |
| 并发控制 | 全局互斥锁，防止并发操作导致状态异常 |

## 页面说明

**抽奖页面** (`/`)：选择奖项 → 点击开始 → 名字滚动 → 点击停止 → 显示中奖结果。只有操作者能看到烟花和中奖弹窗，其他客户端静默更新。

**管理后台** (`/admin.html`)：配置奖项（名称、总人数、单次抽取人数、是否阳光普照），查看参与人员状态，重置抽奖结果。

## 自定义配置

### 修改抽奖名单

编辑 `public/data/list.json`，每条记录包含姓名和部门代码：

```json
{
  "name": [
    { "name": "张三", "dept": "BJ" },
    { "name": "李四", "dept": "SD" }
  ]
}
```

### 替换音乐文件

将 mp3 文件放入 `public/music/` 目录：

- `bg.mp3` — 背景音乐（循环播放）
- `win.mp3` — 中奖音效

没有音乐文件不影响正常使用。

## 公网访问

可通过内网穿透工具实现公网访问：

```bash
# 示例：使用 ngrok
ngrok http 3000
```

## 技术栈

- 后端：Node.js + express + ws
- 前端：原生 HTML + CSS + JavaScript
- 数据持久化：JSON 文件（原子写入 + 备份）
- 实时通信：WebSocket
